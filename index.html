What's a container image?

A "container image" is an executable package of software that includes all of it's dependencies.

Let's create a Docker container image

  • Write a Dockerfile
  • docker build -t fizzbuzz .

What are containers?

  • "A standardized unit of software."
  • A running instance of a "container image."

Let's run our container image

docker run fizzbuzz

How can we share this container image?

By pushing it to Docker Hub!

Let's share this image

docker tag fizzbuzz j3rn/fizzbuzz:latest
docker push j3rn/fizzbuzz:latest

Why use containers?

  • Containers are portable.
  • Containers isolate software from the server environment.
  • Containers are more lightweight than virtual machines.
